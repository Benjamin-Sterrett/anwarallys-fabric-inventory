## 🔴→🟢 PRJ-2960 — P1: QR + label downloads were BROKEN in production (fixed + live-verified)
**Provenance (resolved):** a 3:13 AM Central in-session prompt (transcript `origin: human`) flagged: **"the QR code and label printing is busted, it does not work at all."** Benjamin confirmed he DID type it — late at night; he'd forgotten because Claude was actively smoke-testing at the time. It was NOT based on an independent device test (do NOT record it as one), and it was NOT a Shaaiz complaint (an earlier draft wrongly said "Shaaiz reported"; there is no such message from Shaaiz in the record). What's certain: Claude independently REPRODUCED and PROVED the bug live (not taken on faith). Both Bundle-#3 headline features (Download QR #88, Download label #89) **silently produced nothing** in the browser. Ran the **full debug-workflow**.
- **Root cause (proven live via A/B):** both rasterize the QR by serializing `<QRCodeSVG>` via `renderToStaticMarkup` → load as `<img>`. The serialized `<svg>` had **no `xmlns`**, so the browser refuses to decode a standalone `image/svg+xml` document → `img.onerror` → the SVG→canvas→PNG/PDF pipeline aborts → silent no-op download. Print-QR was fine (renders SVG inline, no rasterization). **Unit tests could not catch it — they mock `Image` so `onload` always fires; jsdom/happy-dom never actually decode SVG.** This is why 2942/2943 shipped "green" but broken.
- **Fix:** shared `serializeQrSvgMarkup()` in `src/lib/labelImage.tsx` guaranteeing `xmlns="http://www.w3.org/2000/svg"` on the root `<svg>` (idempotent), routed both `renderQRImage` + `renderQrPngDataUrl` through it (de-duped 2 identical blocks). Regression test asserts the namespace in the serialized markup — the layer that actually broke.
- **Reviewers:** Semgrep 0 · DeepSeek NO_FINDINGS ×2 · Codex APPROVE ×2. **Merged** PR #90 via **owner_override** (Benjamin-authorized solo self-merge; daemon-signed lead ack). main HEAD now `02f3f86`, deployed, 257/257 tests.
- **LIVE re-smoke on the deployed bundle (`index-BnLm_LEi.js`), SMOKE-001:** Download QR → real `image/png` **14,872 B**; Download label → real `application/pdf` **176,694 B**. Both work now. (Before: nothing.)
- **Lesson banked:** for any raster/canvas/download pipeline, a live browser smoke is the ONLY real proof — mocked-`Image` unit tests are structurally blind to decode failures.
