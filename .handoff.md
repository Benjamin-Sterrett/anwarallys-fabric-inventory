## Locked architecture (full detail in `research/synthesis.md`)

**Data model:**
- Flat top-level collections: `/folders/`, `/items/`, `/movements/`, `/deletedRecords/`
- Folder tree via `parentId + ancestors[]` (IDs, not names — survives renames)
- `folderAncestors[]` denormalized on items for subtree queries
- **No cached `activeItemCount`** — on-demand `count()` aggregation prevents drift
- TS interfaces locked in synthesis §1

**Stock writes:**
- **`runTransaction()` wraps item update + movement creation atomically.** Two writes that can succeed independently = wrong-stock bug.
- Block writes when offline (pilot policy — Wi-Fi confirmed unreliable in storerooms)
- Pessimistic UI: button disabled with spinner until 2xx, then UI updates
- 10-sec timeout surfaces failure dialog regardless

**QR:**
- `qrcode.react` `<QRCodeSVG>`, **Level Q**, SVG only
- Permanent URL `https://<host>/i/{firestoreAutoId}` — **NEVER regenerated** on edit
- Scan landing is **READ-ONLY** — explicit "Adjust stock" button to enter mutation flow
- Default OS camera handles scanning; no in-app scanner

**Auth (UPDATED 2026-04-26 — staff CRUD self-serviced):**
- Email/password + LOCAL persistence (`indexedDBLocalPersistence`)
- **Device-bound login** — sign in once per device, stay logged in, all changes tagged with that device's logged-in user
- **No "Who is doing this?" actor selector** at every change
- **Two roles only:** `admin` (email matches `VITE_ADMIN_EMAIL`) vs everyone else. Both can adjust stock; only admin manages staff via in-app `/staff` page (PRJ-856).
- Bootstrap: Benjamin creates the first admin user (Shaaiz's email) in Firebase Console, sets `VITE_ADMIN_EMAIL` in Cloudflare Pages env. Client logs in and self-services from there.

**UX:**
- 48×48 px min touch targets, 12px spacing
- Hybrid numeric input: "Sold/used" (subtract) vs "Set to exact" (direct), `inputmode="decimal"`, ± stepper at 0.5m
- Drill-down folder nav with breadcrumb chips for depth ≥3, search at depth ≥4
- Confirmation modal: 32px old→new + reason chip + hold-to-confirm 800ms + button label shows transition
- 15s undo snackbar after save (everyone gets it)
- Reason chip selector required before Save enabled (Sold/Cut/Damage/Return/Correction/Receive/Other)
- Traffic-light global sync indicator in app bar (🟢 Online / 🟡 Syncing / 🔴 Error)
- ESL-friendly copy: icon+text labels, Grade 5-6 reading level, no idioms, plain-language errors

## Non-negotiable invariants (Codex enforces; AGENTS.md cites)

- 1 roll = 1 item; **never introduce a separate editable roll count**
- Permanent QR per item — never changes on edit
- Stock adjustments require actor + reason/note + old + new written **atomically** (`runTransaction`)
- Folders support ≥4 levels with **no hardcoded depth logic**
- Movement history is **append-only** (Security Rules enforce)
- 7-day soft-delete window with restore preserves history
- **Firebase Admin SDK NEVER in app path** (breaks Workers runtime)
- All client-side writes must surface success/failure visibly — **never silent fail**
- Errors never silent — pessimistic UI + MD3 dialog on failure + 10-sec timeout
