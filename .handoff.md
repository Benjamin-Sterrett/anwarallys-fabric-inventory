## Locked Architecture (full detail in `research/synthesis.md`)

- **Data:** flat `/folders` + `/items` collections, `parentId + ancestors[]` tree, `folderAncestors[]` denormalized on items, **no cached `activeItemCount`** (on-demand `count()` aggregation)
- **Stock writes:** `runTransaction()` wraps item update + movement creation atomically. Block writes when offline (pilot only).
- **QR:** `qrcode.react` `<QRCodeSVG>`, **Level Q**, SVG only, permanent URL `https://<host>/i/{firestoreAutoId}` — never regenerated on edit
- **Scan landing:** READ-ONLY item view, explicit "Adjust stock" button required to mutate
- **Auth:** email/password + LOCAL persistence; signin method (popup vs Worker proxy) gated by PRJ-806
- **Confirmation modal:** 32px old→new + reason + hold-to-confirm 800ms + button label shows transition + 15s undo snackbar
- **Errors:** pessimistic UI + MD3 dialog for failures + 10s timeout + traffic-light global sync indicator + never silent
