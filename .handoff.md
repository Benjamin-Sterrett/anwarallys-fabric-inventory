## Locked architecture (full detail in `research/synthesis.md`)

**Data model:**
- Flat top-level collections: `/folders/`, `/items/`, `/movements/`, `/deletedRecords/`, **`/users/`**, `/config/`
- Folder tree via `parentId + ancestors[]` (IDs, not names — survives renames)
- `folderAncestors[]` denormalized on items for subtree queries
- **No cached `activeItemCount`** — on-demand `count()` aggregation prevents drift
- TS interfaces locked in synthesis §1; `User` shipped in PRJ-856 PR A

**Stock writes:**
- **`runTransaction()` wraps item update + movement creation atomically.**
- Block writes when offline (pilot policy)
- Pessimistic UI: button disabled with spinner until 2xx
- 10-sec timeout surfaces failure dialog regardless

**QR:**
- `qrcode.react` `<QRCodeSVG>`, **Level Q**, SVG only
- Permanent URL `https://<host>/i/{firestoreAutoId}` — **NEVER regenerated**

**Auth (PRJ-856 + PRJ-871 shipped):**
- Email/password + LOCAL persistence (`indexedDBLocalPersistence`)
- **Device-bound login** — sign in once per device
- **Two roles only:** `admin` (email matches `/config/admin.adminEmail` AND `request.auth.token.email_verified == true`) vs everyone else
- Admin self-services staff via `/staff` page (PRJ-856 + PRJ-871). `createStaffUser` uses a transient secondary Firebase Auth instance so the admin's primary session isn't switched.
- Bootstrap done — see Current Status above.
- **Known parity gap (PRJ-873):** firestore.rules `isAdminUser()` compares email case-sensitively; client `isAdminEmail()` lowercases both sides. Currently mitigated by ensuring `/config/admin.adminEmail` is always lowercase. PRJ-873 fixes durably with `.lower()` in Rules.
- **Known defense-in-depth gap (PRJ-874):** admin can deactivate themself. Client guard shipped in PR #11; Rules guard is the durable fix.

## Non-negotiable invariants

- 1 roll = 1 item; **never introduce a separate editable roll count**
- Permanent QR per item — never changes on edit
- Stock adjustments require actor + reason/note + old + new written **atomically** (`runTransaction`)
- Folders support ≥4 levels with **no hardcoded depth logic**
- Movement history is **append-only** (Security Rules enforce)
- 7-day soft-delete window with restore preserves history
- **Firebase Admin SDK NEVER in app path**
- All client-side writes must surface success/failure visibly — **never silent fail**
- `/users/{uid}` writes go through admin-gated client SDK only (no Admin SDK)
